# webhook 模块

> **最后更新**：2026-04-30

## 职责

接收来自 GitHub 的 Webhook HTTP 请求，完成签名验证、事件路由，将合法事件交给 notification 模块处理。

## 用户旅程

### 旅程 1：PR 事件触发通知

GitHub 发送 Webhook → 服务接收 POST /webhook → 验证 HMAC 签名 → 按仓库名找到对应团队配置 → 将事件和团队配置交给通知派发 → Lark 群收到卡片消息

### 异常旅程

**签名无效**：GitHub 发送伪造/错误签名 → 服务返回 401 → 不触发任何通知 → 攻击者无法注入消息

**仓库未配置**：收到未知仓库的事件 → 静默返回 200 → 不触发通知 → 日志记录 debug

**Lark 发送失败**：网络超时或 Lark 返回 5xx → 服务仍返回 200 给 GitHub（防止重试风暴）→ 错误记录日志
